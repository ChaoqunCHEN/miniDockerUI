# W1-BUILD-STATE-001: JSON Settings Store, Schema Migration, Keychain Abstraction

## Date: 2026-02-22
## Status: completed

## Overview
Implement the persistence layer for application settings: a JSON-backed settings store with schema versioning and migration support, plus a macOS Keychain abstraction for secrets.

## Components

### 1. SchemaVersion (SchemaVersion.swift)
- Semver-style version struct (major.minor.patch)
- Sendable, Codable, Equatable, Comparable, CustomStringConvertible
- Parse from string with validation
- Used by migration system to determine upgrade paths

### 2. JSONSettingsStore (JSONSettingsStore.swift)
- Concrete AppSettingsStore conformance
- Reads/writes AppSettings as pretty-printed JSON
- Returns defaults when file doesn't exist
- Creates parent directories on save
- Extension on AppSettings for currentSchemaVersion and defaultSettings

### 3. SettingsMigrator (SettingsMigration.swift)
- Forward-only migration system
- Chains MigrationStep instances from current to target version
- Rejects downgrades (schemaDowngradeRejected)
- Fails on missing migration path (schemaMigrationUnsupported)

### 4. KeychainStore (KeychainStore.swift)
- Protocol KeychainStoreProtocol for testability
- MacOSKeychainStore concrete implementation using Security framework
- CRUD operations for generic password items
- Idempotent delete (ignores errSecItemNotFound)
